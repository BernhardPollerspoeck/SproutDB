# Admin UI Features — 05.03.2026

## 1. Query History

Das History-Panel zeigt die letzten 50 Queries aus der `_system.audit_log` Tabelle.

- **History-Button** in der Query-Toolbar togglet das Panel auf/zu
- Jeder Eintrag zeigt: farbigen Operation-Badge (GET/UPSERT/DELETE/CREATE), Query-Text (abgeschnitten), Datenbank und relative Zeit
- **Klick auf einen Eintrag** setzt Query + Datenbank im Editor und schliesst das Panel
- **Filter-Input** oben im Panel filtert live nach Query-Text, Operation oder Datenbank
- Panel aktualisiert sich automatisch nach jedem Query-Run

**Dateien:** `QueryPage.razor`, `sproutdb-admin.css`


## 2. Monitoring Sparklines

SVG-Sparkline-Charts zeigen den Verlauf von Metriken als Live-Liniengrafiken.

- **Ring-Buffer** mit 60 Samples (~2 Minuten bei 2s Intervall)
- **Right Panel:** Sparklines unter Writes/s und Reads/s Karten
- **Monitoring-Seite:** Sparklines auf allen 5 Datenkarten (Reads, Writes, Rows, Storage, WAL)
- Reine SVG-Polylines mit halbtransparentem Fill, kein JS noetig
- Updates alle 2 Sekunden zusammen mit den Metriken

**Dateien:** `AdminLayout.razor`, `MonitoringPage.razor`, `sproutdb-admin.css`


## 3. Index Management

Indexe koennen direkt im Schema-Browser erstellt und geloescht werden.

- **IDX-Badge** (blau) erscheint neben indizierten Spalten im Schema-Browser
- **Toggle-Button** pro Spalte (sichtbar bei Hover):
  - Nicht indiziert → Klick erstellt Index (`create index table.column`)
  - Indiziert → Klick loescht Index (`purge index table.column`)
- **Right Panel:** Live-Index-Liste aus `_system.index_metrics` (alle 5s aktualisiert)
  - Zeigt `table.column`, Query-Count und AUTO/MAN Badge
- Fehler bei Index-Operationen werden inline angezeigt

**Backend-Aenderungen:**
- `ColumnInfo.Indexed` Property (bool)
- `TableHandle.IndexedColumns` Property
- `DescribeExecutor` setzt `Indexed` via `table.HasBTree()`

**Dateien:** `ColumnInfo.cs`, `TableHandle.cs`, `DescribeExecutor.cs`, `ResponseHelper.cs`, `SchemaPage.razor`, `AdminLayout.razor`, `sproutdb-admin.css`
